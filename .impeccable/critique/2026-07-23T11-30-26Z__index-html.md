## Design Specificity Verdict

**LLM assessment**: The copy layer is genuinely bespoke — French tutoiement voice, product-specific terms ("Contrat de Constance," "Co-op Milestones," "SOS Restes anti-gaspi"), and a hand-built `calendar-schematic` widget (`styles.css` 223–265) that visually encodes the actual time-blocking pitch. ## Overall Impression

The copywriting and one bespoke widget show real craft and a distinct product voice — but the page currently cannot convert a single visitor (zero clickable elements anywhere in the DOM) and actively misrepresents its product through mismatched stock imagery. The single biggest opportunity: replace the four generic screenshots with even low-fidelity mocks in the `calendar-schematic` treatment, and add a CTA. Fixing just those two would move this from "well-written mood board" to an actual landing page.

## What's Working

1. **The `calendar-schematic` widget** (`styles.css` 223–265, `index.html` 79–95) — a real, hand-authored 7-day grid with two highlighted slots and a caption naming actual times. The only visual on the page that visualizes the specific mechanic being pitched.
2. **Restrained, cohesive visual system** — the `--forest`/`--mint`/`--cream` palette with serif headings over sans body gives a calm, "system not hype" register that matches a positioning built around eliminating chaos.
3. **The Module 04 AI coach quote** (`index.html` 161–166) — specific, warm, written in the product's own voice ("J'ai analysé tes performances au squat cette semaine... tes nuits du jeudi étaient courtes"). The strongest copy on the page and the genuine emotional peak.

## Priority Issues

**[P0] No call-to-action exists anywhere on the page**
- **Why it matters**: Confirmed via full-document search — zero `<button>`, `<a href>`, `<form>`, or `<input>` elements exist. Every persona's only possible outcome is "read and leave." A Persuade-mode page with no ask converts nothing.
- **Fix**: Add a primary CTA in the hero (e.g. "Rejoindre la liste d'attente"), repeat it after the Module 04 emotional peak, and again in the footer, each pointing to a real destination.
- **Suggested command**: `/impeccable harden` (or `/impeccable shape` if the CTA destination/flow itself needs design first)

**[P1] Product imagery is misrepresented as Synk's own UI**
- **Why it matters**: `alt` text on 3 of 5 images explicitly claims unrelated stock screenshots ("James's" fitness home, a generic onboarding flow, "Kevin's" meal-plan app) are Synk's product, and Module 02's image is a cropped sign-in screen from an unrelated branded product ("Doshe"). This fails design specificity and is a factual-misrepresentation risk the moment this page is shown to real users or press.
- **Fix**: Commission or mock actual Synk UI for all four modules — even low-fidelity wireframe screens matching the `calendar-schematic` treatment — or make the imagery abstractly decorative rather than claiming to be product screenshots.
- **Suggested command**: `/impeccable document` then a redesign pass on the module imagery (or `/impeccable polish` once replacement assets exist)

**[P1] No reassurance, proof, or offer near the persuasive climax**
- **Why it matters**: Immediately after the Module 04 emotional peak, the page has no pricing, launch timing, privacy note (relevant given the product wants calendar access), testimonial, or FAQ — it just cools through the Flywheel recap and Avantages cards, then ends on a bare `© 2026 Synk — Concept.` line. Classic peak-end rule violation: the last impression undercuts the peak.
- **Fix**: Add a short reassurance/proof block (e.g. "Gratuit pendant la bêta," a data-privacy note for calendar access, or a testimonial) directly before or as part of a closing CTA section.
- **Suggested command**: `/impeccable clarify` for copy, `/impeccable layout` for placement

**[P2] WCAG contrast failures on accent-green labels**
- **Why it matters**: `.module__eyebrow` (`--accent #3FA34D`, 12.8px bold uppercase) on `--mint #EAF7DD` ≈ 2.88:1 contrast — under the 4.5:1 AA threshold for non-large text. `.hero__wordmark` on the forest gradient ≈ 3.9:1, also under threshold. The mechanical detector did not catch this (clean `[]` result), so it would ship unnoticed without this pass.
- **Fix**: Darken/lighten `--accent` for these specific label uses, or increase size/weight past the WCAG "large text" threshold (≥18.66px bold / ≥24px regular).
- **Suggested command**: `/impeccable audit` (accessibility-focused pass), then `/impeccable polish`

**[P2] No focus states, no landmark structure**
- **Why it matters**: `styles.css` defines zero `:hover`/`:focus`/`:focus-visible` rules, and `index.html` has no `<main>`, `<nav>`, or `<header>` landmark — only bare `<section>`s, no skip-link. Low-impact only because no focusable elements exist yet, but this will bite the moment the P0 CTA is added, since there's no established focus-visible pattern to extend.
- **Fix**: Add landmark elements (`<header>`, `<main>`, `<nav>` if a nav is added) and a `:focus-visible` style before adding any interactive elements.
- **Suggested command**: `/impeccable harden`

## Persona Red Flags

**Jordan (First-Timer)**: Reads the hero promise, scrolls through all four modules and the flywheel recap, reaches the footer — and never encounters anything to click. Jordan cannot tell if Synk is available now, in beta, or just a concept (the footer's "Concept" label is the only hint, and it's the very last word on the page). Jordan has no way to know the module screenshots aren't really Synk's product, since nothing on the page indicates otherwise.

**Riley (Deliberate Stress Tester)**: Tabs through the page with a keyboard — nothing receives focus anywhere, because nothing is focusable, and there's no `:focus-visible` style defined for when something eventually is. Riley checks `alt` text against the actual images and catches the "Synk" branding claims on `assets/2.jpg`, `1.jpg`, `4.jpg` (unrelated apps) and the `Doshe` sign-in screen cropped into `assets/5.jpg`. Riley also flags `.module--reverse .module__grid { direction: rtl }` as a code smell that can diverge visual reading order from DOM/assistive-tech reading order.

**Casey (Distracted Mobile User)**: Sections stack to one column reasonably below the 800px breakpoint, but Casey faces 9 full sections of dense paragraph copy with no progress indicator and no sticky nav/CTA to anchor to. All 5 images load eagerly (no `loading="lazy"`, no declared `width`/`height`), risking layout shift precisely while skimming on a slow connection. Even scrolling all the way down, the payoff is a copyright line, not a reward or next step.

## Minor Observations

- No `<meta name="description">` or Open Graph tags — link previews when shared will be blank/generic.
- No favicon link in `<head>`.
- `crop-decor__piece--star`/`--ribbon` class names (`styles.css` 289–295) reference decorative content (a star, a ribbon) that exists only in the unrelated source image's doodles, not anything native to Synk's brand — the imagery problem is quietly encoded into the codebase's naming itself.
- `html { scroll-behavior: smooth }` currently does nothing, since there are zero anchor links anywhere to scroll to.
- Heading hierarchy inconsistency: module titles use `<h3>` while sibling top-level sections ("Le constat," "Une plateforme, quatre moteurs") use `<h2>`, despite modules being equally top-level, not nested — produces a skipped/inconsistent outline for screen-reader heading navigation.
- No explicit `width`/`height`/`aspect-ratio` on any of the 5 `<img>` tags (only CSS `max-width`), risking layout shift while images load.
- Two inline `style` attributes on `<img>` tags (lines 146, 169) duplicate what could be CSS classes.
